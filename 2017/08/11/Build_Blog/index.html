如何搭建一个简单、实用、免费、美观的个人博客

发表于 2017-08-11

博客技术类型：GithubPages + Hexo,主题采用：Next

搭建个人博客的方法有很多种，至于为什么会选择采用 GithubPages + Hexo 呢？其实文章标题就告诉大家了他的优点。另外的话，也是目前搭建博客的主流方法、使用者众多。

基本搭建

  • Github 创建远程仓库

首先，需要注册 Github 账号（Github 官网链接：https://github.com）。如果你已经有了 Github 账号，那么请忽略这一步。

进入到 Github 主页后，点击右上角的“New repository”，建立一个博客的仓库。切记“Repository name”仓库名称格式一定要：YourName.github.io,也就会你在 Github 上的名称.github.io。其他地方不需要做修改，选择默认即可，然后点击“Greate repository”创建仓库。

  • 本地安装 Git

为什么要安装 Git？因为在本地写的文章要通过 Git 工具来提交到 Github 远程仓库上。

打开 Git 官网（地址：https://git-scm.com），打开“Downloads”页面根据自己电脑版本下载即可。下载完成后安装到本地，安装时不需要修改选项。

  • 本地生成 SSH key & Github 添加 SSH key

SHH 是一种网络协议，用于计算机之间的加密登录。

没有 SSH 授权，我们就不能在本地使用 Git 提交到 Github，所以我们要在本地生成 SSH key，然后再到 Github 添加 SSH key 配置，这样才能达到本地与远程的配对。

Windows 系统在安装 Git 会自带 SSH，我们打开 Git Bash，输入命令：ssh-keygen -t rsa。连续按下三个回车键，不需要输入密码。就会在（C:\Users\username\.ssh）中生成文件 id-rsa 和 id-rsa.pub 两个文件。id-rsa 是密钥，而 id-rsa.pub 是公钥。

接着在 Github 上添加 SSH key，进入“Settings”设置页面，选择“SSH and GPG keys”,点击右上角“New SSH key”来添加 SSH key。将本地的 id-rsa.pub 文件的内容 copy 到 key 那一栏即可，Title 那一栏可以选择不填，最后点击“Add SSH key”就设置成功了。

  • 安装 Nodejs

前面有提到我们是基于 Hexo 来搭建博客，而 Hexo 是一个基于 Nodejs 的静态博客网站生成器。所以我们在使用 Hexo 之前要先安装好 Nodejs。

从官网下载 NOdejs (地址：https://nodejs.org)，使用 LTS 版本。安装过程默认选项即可，不需要做更改。环境变量在安装的过程中已经帮我们自动配置了，无需再次配置环境变量。

  • 安装 Hexo

Hexo 官网（链接：https://hexo.io）

Hexo 的安装就简单了，我们只需要打开 Git Bash，输入命令：npm install hexo-cli -g,按下回车一键安装。

  • 创建本地博客地址

在本地新建你想放博客的文件夹。（例如：F:\Blog）

win + R 输入 cmd 回车打开命令行，定位到本地博客文件夹位置。然后输入命令 hexo init YourName.github.io,这里的名称和 Github 仓库名称保持一致。

这时候在你本地博客文件中，自动生成了一推文件。“source”目录下就存放着你在本地的文章。

配置主题

Hexo 给我们自带了 landscape 主题，如果你不想更换主题，那么走到上一步时，你就基本已经搭建好了。

  • 克隆 Next 主题到本地

打开 Git Bash,定位到你本地博客 YourName.github.io 这个目录。输入命令：git clone https://github.com/iissnan/hexo-theme-next themes/next,这一个命令是什么意思呢？就是将 Github 上开源的 Next 主题克隆到你本地博客位置中的 themes/next 下。

  • 修改配置文件

注意：有两个 _config.yml 文件，一个是本地博客根目录下 Hexo 的配置文件，一个是 themes/next 下主题 next 的配置文件。而且在这两个配置文件中的键值之间是有空格的。

找到本地博客根目录下的 _config.yml 文件，使用文本编辑器打开，将 theme 的值改为 next。关于 Hexo 配置 _config.yml 详情（链接：https://hexo.io/zh-cn/docs/configuration.html）。

主题 Next 中的 _config.yml 的修改（链接：http://theme-next.iissnan.com/getting-started.html#theme-settings）

  • 添加页面

打开 Git Bash，定位到 YourName.github.io 文件夹下，命令：hexo new page categories，这时候我们在博客中的 source 下就新生成了 categories 的文件夹。

  • 本地预览博客

博客的文章存放在 YourName.gihub.io/source/_posts 文件下。文章的格式是 md 格式。

Git Bash 中输入命令：hexo s,表示启动服务器预览。随后可以通过浏览器输入地址 http://localhost:4000 来访问本地博客。

  • 安装自动部署发布工具

Git Bash 中输入命令：npm install hexo-deployer-git –save

  • 将本地博客发布到 GithubPages 上

在 Git Bash 中执行以下命令

命令：hexo clean 表示清除缓存

命令：hexo g 表示生成静态网页

命令：hexo d 表示开始部署

也可以一次性执行：hexo clean && hexo g && hexo d

部署成功后，将可以访问 https://YourName.github.io 查看属于你自己的博客了。
